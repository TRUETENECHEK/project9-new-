# Project Usus: Pipeline for FASTQ Demultiplexing

**Project Usus** — это высокопроизводительный Python-пайплайн для демультиплексирования сырых данных секвенирования (FASTQ) на основе баркодов. Инструмент автоматически распознает универсальные адаптеры, определяет ориентацию ридов (PLUS/MINUS), сопоставляет баркоды и генерирует подробные аналитические отчеты.

## Основные возможности

*   **Гибкий поиск адаптеров:** Использование локального выравнивания для надежного поиска универсальных адаптеров даже в "грязных" данных.
*   **Двусторонняя демультиплексация:** Поддержка комбинаций Forward (FB) и Reverse (RB) баркодов.
*   **Учет ориентации:** Автоматическое определение и коррекция инверсии цепи (поддержка PLUS и MINUS ориентаций).
*   **Тримминг:** Удаление технических последовательностей (адаптеров и баркодов) для получения чистых целевых ридов.
*   **Визуализация и отчетность:** Генерация графиков распределения ридов, статистики ошибок и подробных Excel-отчетов.

## Структура проекта

Пайплайн состоит из следующих ключевых модулей:

1.  **`main.py`** — Точка входа. Координирует процесс: загрузку данных, запуск движка демультиплексирования и генерацию финальных отчетов.
2.  **`demux_core.py`** — "Сердце" системы. Содержит логику поиска адаптеров (`find_anchors`) и подбора лучших баркодов с использованием `Bio.Align.PairwiseAligner`.
3.  **`utils.py`** — Вспомогательные функции для парсинга FASTA-файлов с баркодами и работы с обратно-комплементарными последовательностями.
4.  **`visualization.py`** — Модуль для создания графического отчета (`demux_report.png`), включающего распределение ридов по образцам и статистику ошибок.
5.  **`errors_visual.py`** — Дополнительная визуализация метрик качества выравнивания (процент несовпадений, гэпы, пенальти).

## Входные и выходные данные

### Входные данные (директория `read_file/`):
*   `test_reads.fastq`: Файл с сырыми ридами.
*   `barcodes.fasta`: Файл с последовательностями баркодов (ID должны начинаться с `FB` для прямых и `RB` для обратных).

### Выходные данные (директория `demux_results/`):
*   **FASTQ файлы:** Отдельный файл для каждого распознанного образца (напр., `FB1_RB1.fastq`).
*   **`demultiplexing_report.xlsx`**: Сводная таблица с количеством ридов и средними длинами для каждого сэмпла.
*   **`benchmark_trimmed_sequences.xlsx`**: Детальные данные по каждому риду для оценки качества.
*   **`demux_report.png`**: Визуальный дашборд со статистикой.

##  Как это работает

1.  **Поиск якорей (Anchors):** Алгоритм ищет универсальные Forward и Reverse адаптеры на концах рида в окне 120 п.н.
2.  **Определение цепи:** Если адаптеры найдены в правильном порядке — это цепь PLUS. Если в обратном — MINUS (в этом случае рид переворачивается).
3.  **Идентификация баркодов:** В зонах за адаптерами ищутся наиболее подходящие баркоды из предоставленного списка.
4.  **Тримминг и сохранение:** Последовательность между адаптерами вырезается и сохраняется в соответствующий FASTQ файл.
